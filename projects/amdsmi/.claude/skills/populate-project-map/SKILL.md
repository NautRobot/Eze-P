# Populate Project Map

Creates or updates `.claude/project-map.md` — a single source of truth for the amd-smi project layout. All review agents and skills reference this instead of embedding layout info.

## Steps

1. If `.claude/project-map.md` exists, update it. Otherwise, create it.
2. Explore the actual workspace to discover paths. Double-check that every listed path exists.
3. Identify:
   - Source directories by language (C/C++, Python, Go, Rust, CMake)
   - Build artifacts and packaging scripts
   - Test directories
   - Documentation directories
   - Configuration files (pre-commit, formatters, build configs)
   - Generated files and their generators
   - Critical high-churn files
4. Write the map in the format below.
5. Open for user inspection.

## Format

```markdown
# amd-smi Project Map

> Auto-generated by populate-project-map skill. Single source of truth for project layout.
> Referenced by review agents — update when directory structure changes.

## Source Directories

| Language | Directory | Purpose |
|----------|-----------|---------|
| C/C++ | `src/amd_smi/` | Core library implementation |
| C/C++ headers | `include/amd_smi/` | Public API headers |
| Python bindings | `py-interface/` | `amdsmi_interface.py`, `amdsmi_wrapper.py` |
| Python CLI | `amdsmi_cli/` | CLI tool (`amdsmi_commands.py`, `amdsmi_parser.py`) |
| Go | `goamdsmi_shim/` | Go shim library |
| Rust | `rust-interface/` | Rust bindings |
| CMake | `cmake_modules/` | Build helpers, packaging, versioning |

## Build & Packaging

| Path | Purpose |
|------|---------|
| `CMakeLists.txt` | Root build configuration |
| `RPM/` | RPM post/pre scripts |
| `DEBIAN/` | DEB packaging scripts |
| `pyproject.toml` | pip build config (CLI) |
| `py-interface/pyproject.toml.in` | pip build config (bindings) |

## Tests

| Path | Purpose |
|------|---------|
| `tests/amd_smi_test/` | C++ GTest suite (amdsmitst) |
| `tests/python_unittest/` | Python unit tests |
| `tests/abi_check/` | ABI compatibility checks |

## Documentation

| Path | Purpose |
|------|---------|
| `docs/` | Sphinx documentation source |
| `docs/doxygen/` | Doxygen config for C API |
| `docs/how-to/` | How-to guides |

## Key Tools

| Path | Purpose |
|------|---------|
| `tools/update_wrapper.sh` | Regenerates `py-interface/amdsmi_wrapper.py` (uses Docker + `generator.py`) |
| `tools/generator.py` | Python script that parses C header and emits wrapper — called by `update_wrapper.sh` |
| `tools/update_rust_wrapper.sh` | Regenerates Rust bindings |
| `tools/amdsmi_quick_start.py` | Quick start helper |

## Critical Files (high churn — review carefully)

| File | Risk |
|------|------|
| `amdsmi_cli/amdsmi_commands.py` | CLI behavior regressions, output format |
| `src/amd_smi/amd_smi.cc` | Core C library — correctness, NIC/switch |
| `py-interface/amdsmi_interface.py` | Python API — must sync with C header |
| `include/amd_smi/amdsmi.h` | Public API — cascades everywhere |
| `py-interface/amdsmi_wrapper.py` | Generated bindings + library loader |
| `amdsmi_cli/amdsmi_parser.py` | Argument parsing |
| `CMakeLists.txt` | Build system, packaging, install targets |

## API Cascade Path

Changes to the public C API must propagate through all layers:

`include/amd_smi/amdsmi.h` → `src/amd_smi/amd_smi.cc` → `tools/generator.py` → `py-interface/amdsmi_wrapper.py` → `py-interface/amdsmi_interface.py` → `amdsmi_cli/amdsmi_commands.py` → `docs/`

## Formatter Configuration

| Language | Tool | Config File |
|----------|------|-------------|
| C/C++ | clang-format | `.clang-format` |
| Python | Ruff | `pyproject.toml` |
| CMake | gersemi | `.gersemirc` |
```

## When to Regenerate

- New top-level directories added
- Major restructuring (files moved between directories)
- New language bindings added
- Build system changes (new packaging targets, new CMakeLists.txt files)
